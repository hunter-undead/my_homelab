# Лабораторная работа Мониторинг и анализ трафика (tcpdump + nmap)

## Подготовка пакетов 

- ~$ apt-get install tcpdump nmap -y

## Разбор базовых команды tcpdump

1) ~$ sudo tcpdump -D
- -D показывает список всех сетевых интерфейсов, которые tcpdump может слушать.
[Скриншот](Screen_for_lab/3/1.png)


2) ~$ sudo tcpdump -i (интерфейс)

- -i слушать определенный интерфейс. На экран будет вывод всех пакетов, которые проходят через этот интерфейс.
[Скриншот](Screen_for_lab/3/2.png)

**Чтобы остановить прослушивание Ctrl+C**

3) ЗАХВАТИТЬ ТОЛЬКО ICMP-пакеты (ping)

~$ sudo tcpdump -i ens19 icmp -n

- icmp — фильтр: только ICMP-пакеты (пинг-запросы и ответы)
- -n — не преобразовывать IP-адреса в имена (быстрее и понятнее)
[Скриншот](Screen_for_lab/3/3.png)

4) Захватить DNS-трафик (порт 53)

~$ sudo tcpdump -i ens19 port 53 -n -v

- port 53 — фильтр: только пакеты на 53-й порт (DNS)
- -v — подробный вывод (видно больше деталей)

Для проверки нам нужно с cli сделать dig
~$ dig google.com / ya.ru

[Скриншот](Screen_for_lab/3/4.png)

5) Захватить определённое количество пакетов

~$ sudo tcpdump -i ens19 -c 10 -n

- -c кол-во пакетов, которое хотим перехватить.


6) Сохранить захваченный трафик в файл

~$ sudo tcpdump -i ens19 -c 20 -w /tmp/tcpdump.pcap -n

- -w /tmp/tcpdump.pcap — записать сырые пакеты в файл (бинарный формат PCAP) (Путь может быть абсолютно любой)
- потом эти данные можно открыть в Wireshark или прочитать tcpdump 

7) Прочитать сохранённый файл

~$ sudo tcpdump -r (путь к файлу) -n
[Скриншот](Screen_for_lab/3/5.png)

## Разбор базовых команд nmap

1. Ping-сканирование (найти живые хосты в сети)

~$ nmap -sn 172.16.31.0/27
(Смотрим в мою локальную сеть)

- -sn это ping-scan, не трогаем порты, только проверка того какие хосты отвечают.

[Скриншот](Screen_for_lab/3/6.png)

2. Сканирование портов на конкретном хосте

~$ nmap 172.16.31.1 

Сканирует самые популярные порты на роутере (22-ssh, 53-dns, 80-http, 443 - https и др.)

**Скан для конкретных портов nmap -p 22,53,80 172.16.31.1**

[Скриншот](Screen_for_lab/3/7.png)

3. Определение версий служб

~$ sudo nmap -sV -p 22,53 IP который сканируем

- -sV — определить версии служб, которые висят на открытых портах
- sudo — нужно для глубокого зондирования

[Скриншот](Screen_for_lab/3/8.png)

4. Определение операционной системы

~$ sudo nmap -O 172.16.31.1
