# Настройка на роутере (ALT Srv)
ALT SRV (он же и роутер) - ens18 NAT, ens19 local - 172.16.31.1/27
ALT CLi - получает с DHCP адрес из сети 172.16.31.0/27

## Работа на ALT Srv

1. Устновка необходимых пакетов
- apt-get install bind bind-utils

2. Редактирование файла options.conf
- vim /etc/bind/options.conf
Добавить в слушающий порт 172.16.31.1 (локалка от роутера)
В forwarders вписать 8.8.8.8 и 1.1.1.1
[Скриншот](Screen_for_lab/2/1.png)
3. Перезапускаем службу bind

4. Произведем проверку с CLI
Мы обращаемся к яндексу через IP нашего роутера:
[Скриншот](Screen_for_lab/2/2.png)

Если мы попробуем сделать пинг без указания через какой IP:
[Скриншот](Screen_for_lab/2/3.png)

В результате мы увидим, что используется настроейнный DNS-сервер и он отвечает на запрос.

5. Продолжение настройки на SRV. Настройка BIND в качестве первичного DNS-сервера.
- vim /etc/bind/zone/db.atl.test
[Скриншот](Screen_for_lab/2/4.png)

Проверка правильности настройки зоны:
- named-checkzone mydomain.expansion /etc/bind/zone/db.mydomain.expansion
Если все верно, то выведет, что зона загружена.

6. Отредактируем файл конфигурации зон.
- vim /etc/bind/rfc1912.conf
Добавим свою зону
[Скриншот](Screen_for_lab/2/5.png)

- named-checkconf (если нет вывода -- супер)
Перезагрузим bind.

7. Проверим что DNS-сервер отвечает на запрос по имени домена. (Проверка идет на CLI)
[Скриншот](Screen_for_lab/2/6.png)
