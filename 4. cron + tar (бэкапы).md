# Cron + tar (Бэкап с ротацией)
**Цель.**

Научиться автоматизировать резервное копирование через cron и правильно работать с tar.

**Задание.**
Создать задание в cron, которое:

Каждый день в 03:00 создаёт архив директории /etc/bind

В имени архива указывается текущая дата (например, bind_backup_2025-04-28.tar.gz)

Архивы старше 7 дней автоматически удаляются

## Подготовка файлов для скрипта

1. sudo mkdir -p /backup/bind
Здесь будут храниться наши бэкапы для bind (DNS)

2. sudo vim /usr/local/bin/bind_backup.sh

/usr/local/bin/ -- выбрана для пользовательских скриптов. (Можно любую иную папку, главное были бы права).

Содержимое файла:
#!/bin/bash

BACKUP_DIR="/backup/bind"
SOURCE_DIR="/etc/bind"
DATE=$(date +%Y-%m-%d)
ARCHIVE_NAME="bind_backup_${DATE}.tar.gz"

4. Создаём архив
tar -czf "${BACKUP_DIR}/${ARCHIVE_NAME}" "${SOURCE_DIR}"

5. Удаляем архивы старше 7 дней
find "${BACKUP_DIR}" -name "bind_backup_*.tar.gz" -type f -mtime +7 -delete

**Пояснение к каждой строке:**
1) #!/bin/bash	Шебанг — указывает, что скрипт выполняется в bash
2) BACKUP_DIR="/backup/bind"	Переменная с путём к папке бэкапов
3) SOURCE_DIR="/etc/bind"	Переменная с путём к исходной папке
4) DATE=$(date +%Y-%m-%d)	Записывает текущую дату в формате ГГГГ-ММ-ДД
5) ARCHIVE_NAME="bind_backup_${DATE}.tar.gz"	Имя архива с датой
6) tar -czf ...	Создаёт сжатый архив (c — create, z — gzip, f — file)
7) find ... -name ... -type f -mtime +7 -delete	Ищет файлы старше 7 дней и удаляет их

6. Важно сделать файл исполняемым, выдав права.
sudo chmod +x /usr/local/bin/bind_backup.sh

7. Проверка скрипта вручную, до добавления в cron ("автоматизация").
sudo /usr/local/bin/bind_backup.sh

Проверяем /backup/bind/ -- должен появиться архив бэкапа сегодняшнего дня.

## Добавим задание (скрипт) в cron

1. sudo crontab -e

Выбираем удобный редактор и заполняем конфиг:

**0 3 * * * /usr/local/bin/bind_backup.sh**

Описание команды:

0	Минуты	0 минут (ровно в 3:00)
3	Часы	3 часа ночи
*	День месяца	Любой день
*	Месяц	Любой месяц
*	День недели	Любой день недели
/usr/local/bin/bind_backup.sh	Команда	Запустить скрипт

2. Проверяем, что оно задание добавлено

sudo crontab -l

Выведет список заданий. (нашу строку)

3. Проверка логов cron (найти запись о том, что был создан бэкап по расписанию).

sudo grep "bind_backup" /var/log/syslog

Нужно дождаться, когда это время пройдет (машина должна быть включена; Существует anacron, который может делать такое "догоняя" пропущенные задания ).

sudo nano /etc/anacrontab
вписать:
1 5 bind-backup /usr/local/bin/bind_backup.sh
