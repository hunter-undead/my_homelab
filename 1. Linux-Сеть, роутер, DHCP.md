# Сеть для ВМ внутри Proxmox, маршрутизация, DHCP.

## Что было сделано до работой с ВМками

Настроен Proxmox. Используется домашняя сеть. 
Сетевой адаптер в режиме "Network Bridge", отчего все последующие ВМ будут иметь сеть 192.168.31.0/24

Внутри самого Proxmox закинуты необходимые ISO ОС.

Имеется Linux SRV ( в дальнейшем SRV/srv) и Linux Cli ( в дальнейшем Cli/cli) (обе ВМ Alt Linux).

1. SRV имеет 2 адаптера (1 - выход в инет; 2 - локалка(изолированная))
2. Cli имеет только 1 адаптер (локалка)

## Работа с ВМ

Настройка на SRV.

1. ### Установка IP на адаптер для локалки
1. Переходим в /etc/net/ifaces и создаем директорию для второго адаптера (ens19)

2. Задаем IP для ens19 > echo "172.16.31.5/27" > /etc/net/ifaces/ens19/ipv4address

3. Создаем файл options для адаптера со следующим наполнением:
TYPE=eth
BOOTPROTO=static
DISABLED=no
ONBOOT=yes

4. Перезапуск сети "systemctl restart network" и проверяем, что ens19 получил IP-адрес, что мы указали раннее.
[Скриншот](Screen_for_lab/1/1SRV_static_ens19.png)

2. ### Включение маршрутизации

Можно двумя способами:
1) Способ первый.
- echo "net.ipv4.ip_forward = 1" >> /etc/net/sysctl.conf
- sysctl -p /etc/net/sysctl.conf

2) Способ второй.
- Заходим в файл /etc/net/sysctl.conf
- Находим строчку "net.ipv4.ip_forward = 0" и заменяем 0 на 1.
- Сохраняем файл.
- sysctl -p /etc/net/sysctl.conf

Проверим, что оно включилось:
cat /proc/sys/net/ipv4/ip_forward
[Скриншот](Screen_for_lab/1/2ip_forward.png)

### 3. Настройка NAT (маскарадинг)

- Включение NAT прописыванием iptables.

iptables -t nat -A POSTROUTING -o ens18 -j MASQUERADE

- Сохранение правил, чтобы остались после ребута машины.

iptables-save > /etc/sysconfig/iptables

- Включаем автозагрузку для iptables, чтобы служба поднималась после ребута сама.

systemctl enable --now iptables.service

### 4. Настройка DHCP

1. Установка пакета dhcp-server
2. Скопируем шаблон
cp /etc/dhcp/dhcpd.conf.sample /etc/dhcp/dhcpd.conf
Изменим его под наши настройки.
[Скриншот](Screen_for_lab/1/4dhcp_conf.png)
3. Указываем на какой интерфейс будет передаваться пул DHCP
- nano /etc/sysconfig/dhcpd
- DHCPDARGS=ens19
4. Запуск dhpc и проверяем его статус.
- systemctl enable --now dhcpd
- systemctl status dhcpd
[Скриншот](Screen_for_lab/1/5dhcp_status.png)
